> 2. Key Technical Concepts:
> 
> - **VectorDiscoBlock architecture**: repeating unit of scalar-vector network combining `VectorDiscoConvS2` (W_ss, W_vs, W_sv, W_vv), pointwise skip connections, scalar MLP, and `ScalarVectorProduct`
> - **Levels-as-channels**: stacking K vertical levels as channels instead of batch-flattening `(B*K)`, enabling inter-level weight coupling
> - **Interleaved channel layout**: `[T_0, KE_0, ζ_0, δ_0, T_1, KE_1, ζ_1, δ_1, ..., f]` with `_ch(k, kind) = k * N_KINDS + kind`
> - **Folded weight initialization**: set weights in `(level_out, level_in, kind)` view for readability, then reshape to flat `(K * N_KINDS, ...)` for the actual parameter
> - **Lower-triangular W_sv**: encodes hydrostatic geopotential integration (PGF), where T at level k' only contributes to φ at levels k > k'
> - **W_vs diagonal coupling**: curl (d=0) and divergence (d=1) computed per level, diagonal in level dimension
> - **ScalarVectorProduct sign convention** (matching `PrimitiveEquationsStepper`): `weight[ζ_k_ch, k, 1] = +1.0` for vorticity advection, `weight[f_ch, k, 1] = -1.0` for Coriolis
> - **Float32 noise from wide channels**: processing 4K+1 channels in one reduction vs K×1 introduces ~2 ULP noise (~2×10⁻⁷ absolute), accepted at 5×10⁻³ relative tolerance
> - **DISCO divergence convention**: d=1 component is divergence/gradient, d=0 is curl/vorticity
> - **Horizontal advection gap**: V·∇T requires a dot-product operation not in the current block architecture, kept external

**Why the product rule doesn't work for DISCO operators:**

The DISCO divergence computes:
```
div(sV)_i = Σ_j ψ(i,j) · [cos_β(i,j) · s_j · u_j + sin_β(i,j) · s_j · v_j]
```

The scalar field `s_j` appears at the **source point j** inside the convolution sum. The product rule `V·∇s = div(sV) − s·div(V)` would require factoring `s_i` out at the **output point i**, which is a pointwise operation incompatible with any finite-support convolutional operator. This is not a smoothness or approximation issue — it's a fundamental structural property of convolutions.

Momentum advection sidesteps this entirely via the vorticity form `ζ×V − ∇KE`, which replaces `V·∇V` with curl→scalar-vector product and KE gradient — both operations native to the block. No analogous reformulation exists for `V·∇T`.

**Solution:** `HorizontalAdvection` computes `-V·∇T` directly via the DISCO gradient operator (W_sv) + pointwise dot product, batching all K×n_tracers fields in a single conv forward pass.
