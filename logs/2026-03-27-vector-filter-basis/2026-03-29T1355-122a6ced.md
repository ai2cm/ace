> ## Summary of Canonical Dynamical Core Test Cases for Multi-Level Dry Primitive Equations
> 
> ---
> 
> ### 1. The Most Canonical Test: Jablonowski & Williamson (2006)
> 
> **Reference:** Jablonowski, C. and Williamson, D. L. (2006), *"A baroclinic instability test case for atmospheric model dynamical cores,"* QJRMS, 132, 2943–2975. Also NCAR/TN-469+STR.
> 
> This is the community's *de facto* standard deterministic test for multi-level dry dynamical cores on the sphere. It comes in two flavors: a pressure-coordinate (sigma/eta) version in the original paper, and the Ullrich, Melvin, Staniforth & Jablonowski (UMSJ/Ullrich et al. 2014) modernized version formulated in height (z) coordinates, which is what DCMIP2016 uses and for which working Fortran code is openly available.
> 
> ---
> 
> #### Two-Part Test Strategy
> 
> **Part 1 — Steady-state test:** Initialize with the balanced analytic solution (no perturbation). A perfect model maintains it indefinitely; numerical errors cause drift. Run without diffusion. Error norms are computed against the analytic solution. This reveals advection errors, pressure-gradient errors, and conservation defects.
> 
> **Part 2 — Baroclinic wave test:** Add a small-amplitude localized perturbation to the zonal wind to seed baroclinic instability. The wave grows observably by ~day 4 and reaches explosive cyclogenesis by day 8–9. No analytic solution exists for this phase; results are compared against high-resolution reference solutions.
> 
> ---
> 
> #### Initial Condition Formulas (DCMIP2016 / UMSJ Height-Coordinate Version)
> 
> This is the version with a working open-source Fortran implementation at [github.com/ClimateGlobalChange/DCMIP2016](https://github.com/ClimateGlobalChange/DCMIP2016).
> 
> **Physical constants:**
> - Earth radius: a = 6,371,220 m
> - Gravity: g = 9.80616 m/s²
> - Dry gas constant: Rd = 287.0 J/kg/K
> - Rotation rate: Omega = 7.29212e-5 s⁻¹
> - Reference surface pressure: p0 = 100,000 Pa
> 
> **Test case parameters:**
> | Parameter | Value | Description |
> |-----------|-------|-------------|
> | T0E | 310 K | Temperature at equatorial surface |
> | T0P | 240 K | Temperature at polar surface |
> | T0 | 275 K | = 0.5*(T0E+T0P) |
> | B | 2 | Jet half-width parameter |
> | K | 3 | Jet width/power parameter |
> | lapse (Gamma) | 0.005 K/m | Temperature lapse rate |
> | u0 | 35 m/s (implicit, derived) | Maximum zonal wind |
> | up (pertup) | 1.0 m/s | Perturbation amplitude |
> | pertlon | pi/9 = 20° E | Perturbation longitude |
> | pertlat | 2*pi/9 = 40° N | Perturbation latitude |
> | pertz | 15,000 m | Vertical taper cap height |
> | pertexpr | 0.1 Earth radii | Perturbation decay radius |
> 
> **Temperature and pressure (given height z, latitude phi):**
> 
> Define the scale height H = Rd*T0/g and scaled height s = z/(B*H):
> 
> ```
> tau1(z) = (1/lapse) * (lapse/T0) * exp(lapse*z/T0)
>         + ((T0-T0P)/(T0*T0P)) * (1 - 2*s^2) * exp(-s^2)
> 
> tau2(z) = 0.5*(K+2) * ((T0E-T0P)/(T0E*T0P)) * (1 - 2*s^2) * exp(-s^2)
> ```
> 
> ```
> inttau1(z) = (1/lapse) * (exp(lapse*z/T0) - 1)
>            + (T0-T0P)/(T0*T0P) * z * exp(-s^2)
> 
> inttau2(z) = 0.5*(K+2) * (T0E-T0P)/(T0E*T0P) * z * exp(-s^2)
> ```
> 
> Interior temperature factor:
> ```
> I_T(phi) = cos(phi)^K - K/(K+2) * cos(phi)^(K+2)
> ```
> 
> Temperature:
> ```
> T(phi, z) = 1 / (tau1(z) - tau2(z) * I_T(phi))
> ```
> 
> Pressure (hydrostatic):
> ```
> p(phi, z) = p0 * exp[-g/Rd * (inttau1(z) - inttau2(z) * I_T(phi))]
> ```
> 
> **Zonal wind** (geostrophic + centripetal balance):
> ```
> U(z, phi) = (g/a) * K * inttau2(z) * [(cos phi)^(K-1) - (cos phi)^(K+1)] * T(phi, z)
> 
> u_ref(phi, z) = -Omega*a*cos(phi) + sqrt((Omega*a*cos(phi))^2 + a*cos(phi)*U(z,phi))
> ```
> 
> Meridional wind: v = 0. Vertical velocity: w = 0. Surface pressure: ps = p0 = constant (no topography). Surface geopotential: Phi_s = 0.
> 
> **Perturbation** (exponential bell, added to u):
> ```
> r(lon, lat) = (a/pertexpr) * arccos[sin(pertlat)*sin(lat) + cos(pertlat)*cos(lat)*cos(lon-pertlon)]
> 
> Z_p(z) = 1 - 3*(z/pertz)^2 + 2*(z/pertz)^3   for z <= pertz, else 0
> 
> u'(lon, lat, z) = pertup * Z_p(z) * exp(-(r/a)^2 / pertexpr^2)  if r/a < 1, else 0
> ```
> 
> (The stream function perturbation variant uses a cosine bell instead.)
> 
> ---

> #### Original Pressure-Coordinate (JW2006) Version
> 
> The original 2006 paper uses eta = p/ps as the vertical coordinate (sigma). Key formulas use:
> - eta_v = 0.5*(eta - eta_0)*pi, with eta_0 = 0.252 as the jet reference level

> ### 2. Other Standard Benchmarks
> 
> #### Held-Suarez (1994)
> 
> **Reference:** Held, I. M. and Suarez, M. J. (1994), *"A Proposal for the Intercomparison of the Dynamical Cores of Atmospheric General Circulation Models,"* BAMS, 75, 1825–1830. [PDF at GFDL](https://www.gfdl.noaa.gov/bibliography/related_files/ih9401.pdf).
> 
> - **Purpose:** Long-term (1200-day) climate statistics test; dry core forced with simplified physics
> - **Forcing:** Newtonian relaxation of temperature to a prescribed zonally symmetric radiative equilibrium profile; Rayleigh drag on low-level winds (boundary layer friction)
> - **Initial state:** Any reasonable state; the model is run to statistical equilibrium
> - **Variables:** All prognostic variables (T, u, v, ps)
> - **Diagnostics:** Zonal-mean time-mean temperature, zonal wind, eddy kinetic energy, eddy heat and momentum fluxes; compared between models in statistical steady state
> - **Use:** Does not test initial-value accuracy; tests the model's climate in the presence of simplified forcing; widely used to check that a new dynamical core produces a realistic general circulation
> 
> #### Williamson et al. (1992) Shallow Water Test Suite
> 
> **Reference:** Williamson, D. L. et al. (1992), *"A standard test set for numerical approximations to the shallow water equations in spherical geometry,"* JCP, 102, 211–224.
> 
> - **Purpose:** Seven canonical 2D (horizontal only) test cases for the shallow water equations on the sphere, used as a first check before implementing a 3D core
> - **Test cases include:**
>   - Advection across the poles
>   - Steady-state geostrophic flow (global and local scale)
>   - Rossby-Haurwitz waves (wavenumber 4) — exact solution to the non-divergent barotropic vorticity equation; expected to evolve nearly steadily
>   - Zonal flow over a mountain
>   - Observed atmospheric states
> - **Known issue:** The Rossby-Haurwitz wavenumber-4 wave is dynamically unstable (Thuburn & Li 2000); results are model-dependent because numerical truncation errors excite the instability at different rates. This makes it less reliable as a convergence test, though it is still widely used as a qualitative check.
> 
> #### DCMIP Test Suites (2008, 2012, 2016)
> 
> The Dynamical Core Model Intercomparison Project organized by Jablonowski and collaborators has produced three generations of standardized test case documents with open-source Fortran initialization routines:
> 
> - **DCMIP 2012:** 3D tracer transport tests, Held-Suarez, baroclinic wave, and orographic tests; focus on hydrostatic models. Document: Ullrich et al. (2012).
> - **DCMIP 2016:** Moist baroclinic wave (Test 1-1, described above), idealized tropical cyclone (1-2), supercell (1-3), moist Held-Suarez (2-1). Focus on non-hydrostatic models and physics-dynamics coupling. Fortran code: [github.com/ClimateGlobalChange/DCMIP2016](https://github.com/ClimateGlobalChange/DCMIP2016).
>

> Where do we go from here? Is the isobaric assumption the next to relax, if that is relevant to correct JW behaviour?

Good question to ask before investing more effort. Let me give an honest diagnosis.

## What's actually limiting the JW simulation

**The biggest problem is not the isobaric assumption per se — it's `phi_surface = constant`.**

In our model:
- `φ_0 = phi_surface` (spatially uniform, we set it to 0)
- Therefore `∇φ_0 = 0` always
- Level 0 has no pressure gradient force, ever — regardless of what temperature does

This makes the bottom level completely dynamically inert. In the JW test, the baroclinic instability is most energetic near the surface (it's a warm-core surface cyclone / cold-core surface anticyclone pattern), so silencing the surface level is a significant flaw.

**Why does isobaric lead to this?** In isobaric coordinates, the surface is at a fixed pressure (e.g. 1000 hPa), and the "surface geopotential" `φ_s` is what varies to encode surface pressure variations. In our model `φ_s` is a constant because we never evolve it. In sigma coordinates, the prognostic variable is surface pressure `p_s(λ, φ, t)`, which varies in space and time and directly gives spatially-varying `φ_s`. So sigma effectively unlocks what isobaric can't express with a scalar `phi_surface`.

## Full list of limitations, in order of impact on JW

| # | Issue | Impact on JW | Fix |
|---|-------|-------------|-----|
| 1 | `phi_surface` is a spatial constant | Kills surface level dynamics entirely | Sigma coords or prognostic `p_s` |
| 2 | Only 5 levels | Coarse vertical resolution; Eady growth rate is sensitive to vertical wind shear | 15–26 levels |
| 3 | No horizontal diffusion | Energy builds at small scales; by day 10+ the run is increasingly turbulent | ∇⁴ or spectral truncation |
| 4 | Initialization imbalance | Generates gravity wave transient in the first 2–3 days that modulates wave growth | Proper balance init (minor) |

## Is sigma the right next step?

That depends on what you want this model *for*:

- **If the goal is a correct JW reference**: sigma (or hybrid σ–p) coordinates are the right fix. It changes `∂p_s/∂t = -∫∇·(p_s V) dσ` (a new prognostic equation), the pressure gradient force becomes `∇φ + RT/p · ∇ln(p_s)` (an extra term in the momentum equation), and omega becomes more complex. It's a fairly significant restructuring.

- **If the goal is to validate the new adiabatic term and explore multi-level dynamics**: a much cheaper win is to drop to 3–4 active levels but **exclude the surface level** from the dynamics — i.e. place level 0 at 850 hPa (not 1000 hPa) and set `phi_surface` to the analytic JW balanced geopotential at 850 hPa. Then all levels are active. You'd lose the surface-level wave but gain a model that's physically self-consistent.

- **If the goal is eventually to use this as a physics-based emulator backbone**: sigma is the right architecture, but the work is substantial and you'd probably also want flux-form advection and spectral diffusion.

My honest read: the adiabatic term change we just made is correct and necessary; the model now conserves potential temperature following the flow. But for a recognizable JW baroclinic wave you need at minimum to address item 1 (sigma or a non-trivial `phi_surface`). Items 2 and 3 are needed for quantitative agreement with the reference. They're independent work streams and none is trivial.

## Hybrid Sigma-Pressure Coordinate Formulation

**Pressure at each level:**
```
p_k(λ,φ,t) = a_k + b_k · p_s(λ,φ,t)
```
`a_k` (Pa) and `b_k` (dimensionless) are prescribed constants. At the surface interface `b=1, a=0`; at the top interface `b=0, a=p_top`. The pure-sigma case is `a=0, b=σ_k`.

**Layer thicknesses at interfaces:**
```
Δa_k = a_{k+1/2} - a_{k-1/2}    (constant)
Δb_k = b_{k+1/2} - b_{k-1/2}    (constant)
Δp_k = Δa_k + Δb_k · p_s         (B,K,H,W — varies in space)
```

---

### 1. Surface pressure tendency

```
∂p_s/∂t = −∑_k ( Δp_k div_k  +  Δb_k V_k·∇p_s )
```

Reduces to the sigma formula when `Δa=0, Δb=Δσ, Δp=Δσ·p_s`.

---

### 2. η̇ recurrence (vertical velocity diagnostic)

Define `C_{k+1/2} = (∂p/∂η) η̇` at interfaces. Starting from `C_{−1/2} = 0` (top BC):

```
C_{k+1/2} = C_{k-1/2}  −  Δb_k ∂p_s/∂t  −  Δp_k div_k  −  Δb_k V_k·∇p_s
```

Mid-level: `C_k = ½(C_{k-1/2} + C_{k+1/2})`.

Note: the existing `SigmaCoordinateStepper._sigma_dot` is missing the `Δb_k ∂p_s/∂t` term (equivalent to `Δσ_k ∂p_s/∂t`). This is a bug — small but worth fixing in the hybrid version from the start.

---

### 3. Pressure gradient force

```
PGF_k = −∇φ_k  −  (R T_k b_k / p_k) ∇p_s
```

In sigma: `b_k/p_k = σ_k/(σ_k p_s) = 1/p_s`, so the correction becomes `−RT ∇(ln p_s)` ✓.

The geopotential `φ_k` is now a full spatial field because `ln(p_{k-1}/p_k) = ln((a_{k-1}+b_{k-1}p_s)/(a_k+b_k p_s))` varies with `p_s`. Apply the DISCO gradient operator to the resulting (B,K,H,W) field — no structural change needed.

---

### 4. Temperature tendency

The pressure velocity `ω_k = dp/dt`:
```
ω_k = b_k(∂p_s/∂t + V_k·∇p_s) + C_k
```

Temperature tendency:
```
∂T/∂t = −V·∇T  +  (κT/p)ω  −  C_k ∂T/∂p
```

Here `∂T/∂p` uses central finite differences in pressure-space:
```
∂T/∂p|_k = (T_{k+1} − T_{k-1}) / (p_{k+1} − p_{k-1})
```
where `p_{k±1} = a_{k±1} + b_{k±1} p_s` is a spatially-varying field. Similarly for `∂V/∂p` in the momentum vertical advection and `∂q/∂p` in humidity.

In sigma this reduces correctly: `b/p = 1/p_s`, `ω = (∂p_s/∂t + V·∇p_s)/p_s · p + C`, and everything matches.

---

### 5. Hydrostatics

```
φ_0 = φ_surface
φ_k = φ_{k-1}  +  R T_{k-1}  ln(p_{k-1}/p_k)
```

Unlike sigma, `ln(p_{k-1}/p_k)` is no longer a constant — it's a (B,H,W) field computed each step from `p_k = a_k + b_k p_s`.
